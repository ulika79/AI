🚀 旗舰文本模型
GPT-5.6-Sol DeepSeek-V3.2 Qwen3.5-Plus GLM-5.2 Claude-Opus-4-8 LLaMA-3.1-405B Grok-4.5
✅ 优点
  • 顶尖推理能力，复杂任务处理准确
  • 超大上下文（100K~1M tokens）
  • 多模态输入（图像/音频/视频）

  • 成本高昂（尤其 Ultra/Opus 级别）
  • 推理延迟较高
  • 部署需大量算力
🎯 用途 复杂业务推理、科研辅助、法律/医疗分析、高质量长文创作、企业级 Agent
⚡ 轻量/性价比文本
GPT-5.6-Luna Qwen3-0.6B Gemma-7B LLaMA-3.2-3B GPT-4.1-Nano Qwen3-8B
✅ 优点
  • 极低成本（输入 $1/1M tokens）
  • 响应速度极快（<100ms）
  • 可部署于边缘/移动设备
  • 适合高并发大批量任务
❌ 缺点
  • 复杂推理能力弱
  • 多模态能力有限
  • 上下文窗口较小
🎯 用途 大规模分类、情感分析、RAG 前置检索、实时客服、数据预处理
🧮 推理/思维链模型
o1 / o3 / o4-Mini DeepSeek-R1 QwQ-32B Qwen3-*-Thinking
✅ 优点
  • 数学/逻辑/编程推理显著强
  • 思维链可见，可解释性高
  • 在 AIME/MATH 等基准优异
❌ 缺点
  • 推理耗时较长
  • 成本高于普通模型
  • 简单任务“过度思考”
🎯 用途 数学题求解、竞赛编程、复杂规划、策略生成、科研假设验证
💻 代码专用模型
GPT-5.x-Codex Qwen-Coder-Plus DeepSeek-Coder Kimi-K2.7-Code
✅ 优点
  • 专项训练，代码生成/修复强
  • 支持长上下文（40万+ tokens）
  • 多语言多框架支持
❌ 缺点
  • 通用对话能力相对弱
  • 非代码任务效果一般
🎯 用途 自动代码生成、代码审查、单元测试、代码重构、技术文档
🎨 图像生成模型
GPT-Image-2 DALL-E 3 Wan2.7-Image-Pro Qwen-Image-Edit-Plus Grok-Imagine-Image
✅ 优点
  • 高分辨率（2K~4K）
  • 精准控制（文字渲染、身份一致）
  • 强大编辑能力（局部重绘、风格迁移）
❌ 缺点
  • 旗舰模型成本高
  • 人脸/版权内容受限
  • 速度与质量需权衡
🎯 用途 商业设计、广告素材、电商产品图、漫画绘本、概念艺术
🎬 视频生成模型
Vidu Q3-Pro Kling 3.0-Turbo HappyHorse 1.1 Pixverse 系列 Grok-Imagine-Video
✅ 优点
  • 分辨率最高 4K
  • 多镜头、多主体一致性
  • 运动控制、表情迁移、视频延长
❌ 缺点
  • 生成时间长（分钟级）
  • 复杂场景物理逻辑可能出错
  • 音视频同步仍在优化
🎯 用途 短视频/广告制作、虚拟角色动画、游戏预览、教育培训视频
👁️ 视觉理解 (VLM)
Qwen-VL-Max/Plus LLaMA-3.2-11B/90B-Vision GLM-4.6V Claude-Opus-4-5
✅ 优点
  • 图文理解、图表推理、OCR 强
  • 支持高分辨率图像
  • 部分支持视频帧理解
❌ 缺点
  • 参数大，推理成本高
  • 对模糊图像鲁棒性一般
🎯 用途 文档数字化、票据识别、医疗影像辅助、电商图片理解、视觉问答
🎤 语音 / TTS / ASR

📌 注意事项

  • 日期后缀（如 -2026-07-09）表示特定快照版本，优先使用 -latest 或最新日期。
  • 同名不同后缀（如 gpt-5.6-sol vs gpt-5.6-sol-ultra）表示同一模型的不同运行模式（ultra 开启多代理协作）。
  • 旧模型淘汰：gpt-4-turbo-preview、gpt-3.5-turbo-1106、davinci-002 等已被新系列替代，不建议在新项目中使用。
  • 拼写提示：列表中的 viduq1 通常指生数科技 Vidu Q1 的简写。
  • MJ 系列：mj_* 工具需配合 Midjourney 主服务使用，非独立模型。
